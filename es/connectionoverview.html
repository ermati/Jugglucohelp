Mirror

Enviar o recibir datos desde otro dispositivo. Puedes conectarte a un dispositivo Android ejecutando Juggluco para mostrar datos de la misma manera que aquí. Después de recibir los datos de escaneo, el dispositivo espejo también puede contactar con el sensor a través Bluetooth. Para asegurarse de que solo uno de los dispositivos haga contacto con el sensor, "Sensor via Bluetooth" se apaga automáticamente cuando los valores llegan a través de IP/TCP.

En esta pantalla, después wlan, se muestra la IP WLAN local de este dispositivo, usted debe especificar esto en el otro dispositivo de su red doméstica que esté conectando (o su módem si se conecta de forma remota).

Después de Puerto, Puede cambiar el puerto TCP en el que esta aplicación escucha las conexiones.

Haciendo click Añadir conexion puede especificar con qué hosts conectarse y como conectar.

Una conexión de red fiable en Android es interrumpida por muchos tipos de optimizaciones de batería y restricciones de fondo, presionando Modo dormirayuda a cambiar esto.

Si juggluco funciona como una aplicación Mirror (recibe valores de glucosa de otro dispositivo que ejecuta Juggluco), puede presionarAlarmas para llegar a un cuadro de diálogo donde puede especificar alarmas de glucosa alta y baja y datos relacionados con notificaciones.

Espeajo envía acantidades: Si la aplicación recibe cantidades de medicamentos, alimentos y datos deportivos de esta aplicación ejecutándose en otro dispositivo, no deberías agregar o modifique esta misma información aquí. Comprobar esto hace que sea imposible.

Sincronizar envía datos nuevos a un dispositivo conectado si están disponibles o solicita datos nuevos.

Reiniciar: Cierra y crea una nueva conexión.

Wifi (solo en WearOS): Cuando no está configurado, WIFI se apaga automáticamente después de algunos tiempo por WearOS para preservar la duración de la batería y Juggluco cambia al modo uso de Bluetooth para la conexión entre el teléfono y el reloj. Cuando está activado, se le pide a WearOS que mantenga el WIFI activado y tardará más antes de que se apague, pero eso aún puede suceder. Configurando esta opción normalmente no es necesaria y los creadores de WearOS piensan que activar WIFI provocará grandes consumos de la batería.

Conexiones existentes se muestran en una lista, la etiqueta, el puerto IP y el mes, día y hora, los datos de la última sincronización exitosa.

Como se utiliza la conexión se abrevia de la forma siguiente:

n: cantidades (números)

s: scans

b: stream (bluetooth)

r: receive


For more information on Mirror connections read the help under "Add connection" and see https://www.juggluco.nl/Juggluco/mmol.html#mirror

Desktop computer

You can run the Android Juggluco app on a Windows, Macintosh or Linux computer with the help of an Android Emulator (for example Genymotion). The Juggluco on the emulator receives the data via a Mirror connection from an Android phone connected to the sensor. Pinch movements (moving two fingers to or from each other) can be simulated by pressing a key (Control in Genymotion) while moving the mouse pointer.
