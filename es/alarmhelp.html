Alarmas

Alarmas de glucosa

Los valores de flujo recibidos a través de Bluetooth pueden utilizarse para establecer alarmas de glucosa baja y alta.

Alarma glucosa baja:

Valor por debajo del cual debe saltar una alarma.

Alarma glucosa alta:

Valor por encima del cual debe saltar una alarma.

Alarma de perdida de señal:

Especifique después de cuántos minutos debe sonar una alarma si no se recibe ningún valor de glucosa a través de Bluetooth.
Las alarmas se detendrán cuando se cumpla una de las siguientes condiciones:

- La duración especificada ha transcurrido;

- Se toca la vista de curva;

- La notificación Android de Juggluco está tocada (en estado desbloqueado);

- Se toca el botón Cancelar alarma en la notificación de Juggluco.

Valor disponible:

A veces el sensor no funciona. Cuando actives esta opción, recibirás una notificación cuando el sensor vuelva a funcionar, pero sólo cuando hayas visto que falta el valor en la propia app de Juggluco. Cuando sólo sepas que faltaba el valor por la notificación de Juggluco o su esfera de reloj, no se evocará la notificación de valor disponible.
Para cada alarma puedes especificar qué tono de llamada reproducir y durante cuánto tiempo.

Alarma es alarma:

Cuando se configura, es el tono de llamada que se reproduce para las alarmas baja, alta y perdida de señal dadas en la categoría Alarmas de Android. Esto significa que en los ajustes de Android su volumen se gestiona con el nivel de volumen de Alarmas y no se silencia cuando se desactiva el sonido de todo el teléfono o se establece en vibración. Cuando la alarma no está configurada, se utiliza el nivel de volumen del tono de llamada. Este es siempre el caso para el valor disponible notificación.
